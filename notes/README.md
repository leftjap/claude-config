# notes 운영 규칙

## 용도
AI 세션에서 나온 설계 판단, 인사이트, 아이디어를 기록한다.
playbook.md에 넣기엔 무겁고, 버리기엔 아까운 내용.

## 파일명
YYYY-MM-DD_키워드.md

## 문서 구조
```
# 제목

- date: YYYY-MM-DD
- tags: 쉼표로 구분

## 맥락
왜 이 얘기가 나왔는가

## 결론
뭘 결정했는가 / 뭘 깨달았는가

## 후속
이걸로 뭘 할 건가
```

## 운영 원칙
- 폴더는 notes/ 하나로 시작. 파일 20개 넘으면 하위 폴더 분리.
- playbook.md 변경 이력에는 한 줄 요약 + 파일 경로만 남긴다.
- AI가 리마인드 요청을 받으면 notes/ 폴더를 조회해서 tags/제목 기준으로 찾는다.
- 세션 중 "저장"이라고 하면 이 규칙에 따라 문서를 생성한다.
